# clawbot 交付清单

> 目标：明确当前 `localBridge/clawBotCli` 在重构后的交付边界，方便复制到其它工程、交给 AI 继续开发，或作为独立类库继续演进。

---

## 一、核心交付物

以下内容构成当前阶段最重要的交付物：

### 1. 类库主体

```text
clawbot/
```

说明：
- 这是未来真正的核心产品主体
- 对外统一入口：`from clawbot import ClawBotClient`
- 新功能应优先基于这里扩展

### 2. 使用示例

```text
examples/
```

说明：
- 供开发者和 AI 理解如何组合能力
- 示例优先展示新类库用法，而不是旧 `utils/`

### 3. 设计与迁移文档

```text
README.md
重构实施清单.md
最小复制包说明.md
```

说明：
- `README.md`：项目说明与使用入口
- `重构实施清单.md`：重构目标、目录设计、实施策略
- `最小复制包说明.md`：复制到其它工程时的最小边界说明

### 4. 基础依赖

```text
requirements.txt
```

当前最小依赖：
- `requests`

## 三、可选复制内容

以下内容可按需要复制：

### 1. `tests/integration/`
适合：
- 想在目标工程里保留 smoke test
- 想验证目标环境是否能连通 LocalBridge / X / AI 平台

### 2. `tests/unit/`
适合：
- 想保留本地单元测试能力
- 想继续演进媒体上传、兼容层等逻辑

### 3. `scripts/`
适合：
- 想保留 demo 或临时工具脚本

---

## 四、不建议作为核心交付物依赖的内容

### 1. `utils/`
定位：
- 兼容层
- 迁移层
- 旧脚本和旧测试的过渡适配

建议：
- 可以暂时保留
- 但不应再作为新功能开发主入口

### 2. 老的 `tests/*.py`
定位：
- legacy test scripts
- 迁移过程中保留的历史脚本

建议：
- 不作为未来类库设计的核心参考来源
- 已迁出的能力优先看 `examples/` 和 `tests/integration/`

### 3. `openclaw.py`
定位：
- demo / workflow script

建议：
- 可保留为示例
- 但不应视作类库核心 API 定义位置

## 五、给 AI 的开发约束建议

如果把这套代码交给 AI 继续开发，建议明确以下规则：

1. 新功能优先放到 `clawbot/` 下
2. 多步骤组合逻辑优先放到 `clawbot/workflows/`
3. 面向用户的示例放到 `examples/`
4. 自动化验证放到 `tests/unit/` 或 `tests/integration/`
5. 不要继续在 `utils/` 上叠加新能力
6. 不要把新的正式接口继续做成“测试脚本入口”

## 七、media/task 子系统异常约定

对外工程如果使用：

- `clawbot/upload/task_client.py`
- `clawbot/services/media.py`

建议按以下异常语义集成：

### 1. 可预期异常类型

- `ParseError`
  - 配置 JSON 非法
  - 任务创建响应缺少 `taskId`
  - 任务状态响应缺少 `state`
  - 任务结果不是合法 JSON

- `MediaUploadError`
  - 本地文件不存在
  - 任务执行进入 `failed` / `cancelled`
  - 任务结果缺少 `mediaId`
  - 其它上传流程中的未知异常被统一包装后抛出

- `TaskTimeoutError`
  - 任务轮询超时
  - 超时前会尝试取消任务

### 2. 服务层传播规则

- `MediaService.upload()` 会保留 `ParseError` / `MediaUploadError` / `TaskTimeoutError` 的原始语义
- `MediaService.post_tweet()` 与 `reply_with_media()` 在上传失败时不会继续调用动作层
- `MediaService.post_tweet()` 与 `reply_with_media()` 不会吞掉动作层异常，动作异常会直接继续向上抛出

### 3. 外部工程接入建议

推荐调用方显式区分：

1. `TaskTimeoutError`：重试或人工兜底
2. `ParseError`：排查接口契约或配置文件
3. `MediaUploadError`：作为上传业务失败处理
4. 动作层原始异常：作为发帖/回复动作失败处理

## 八、当前交付状态判断

目前已经达到：

- 有独立类库骨架
- 有统一入口 `ClawBotClient`
- 有 examples 可供 AI/开发者参考
- 有 unit/integration 测试结构
- 有 utils 兼容层承接旧脚本
- 有迁移说明和复制说明文档

当前还未完全完成的部分：

- 旧 tests 尚未完全退场
- 部分 legacy 脚本仍在并存
- 兼容层仍存在，后续仍可继续收缩

所以当前状态可以定义为：

**“可交付的迁移版类库”**

即：
- 已可复制使用
- 已可作为 AI 二次开发底座
- 但仍处于从旧结构向新结构收束的过程中

---

## 八、一句话总结

当前阶段真正应该被交付、复制、扩展和继续演进的，是 `clawbot/` 这一套新类库体系；其余目录按“示例 / 测试 / 兼容 / 历史迁移”理解即可。
