# clawbot 最小复制包说明

> 目标：把 `localBridge/clawBotCli` 中已经重构出的类库能力，最小成本复制到其它工程中使用，并让 AI 继续在该类库上开发新功能。

---

## 一、最小必需目录

如果你只想把类库复制到其它工程，最小必需内容是：

```text
clawbot/
requirements.txt
```

其中：

- `clawbot/`：核心类库代码
- `requirements.txt`：当前至少需要 `requests`

如果目标工程已经自己管理依赖，也可以只复制：

```text
clawbot/
```

然后在目标工程依赖里确保安装：

```bash
pip install requests
```

---

## 二、推荐一并复制的目录

如果希望让 AI 更容易理解如何继续扩展，建议一并复制：

```text
clawbot/
examples/
重构实施清单.md
最小复制包说明.md
```

原因：

- `clawbot/`：真正可用的类库主体
- `examples/`：给 AI 和开发者看的组合示例
- `重构实施清单.md`：帮助 AI 理解当前类库设计目标与后续演进方向

---

## 三、不建议作为最小包复制的内容

以下目录目前更多是迁移期产物，不建议作为最小可用类库的一部分强依赖：

- `tests/`
- `utils/`
- `openclaw.py`
- `scripts/`

说明：

- `utils/` 当前主要是兼容旧接口
- `tests/` 主要是验证与迁移过程的产物
- `openclaw.py` / `scripts/` 更像示例或 demo

---

## 四、复制到其它工程后的推荐结构

```text
other_project/
├── app/
├── clawbot/
├── main.py
└── requirements.txt
```

然后在业务代码中：

```python
from clawbot import ClawBotClient

client = ClawBotClient()
print(client.x.status.is_logged_in())
```

---

## 五、推荐给 AI 的扩展入口

当 AI 在目标工程中继续开发时，优先从这些入口扩展：

- `client.x.status`
- `client.x.timeline`
- `client.x.tweets`
- `client.x.users`
- `client.x.search`
- `client.x.actions`
- `client.media`
- `client.ai.chat`
- `client.workflows`

这些入口已经比旧的 `utils/api_client.py` 更适合作为稳定基础能力层。

---

## 六、最小调用示例

### 1. 查询状态

```python
from clawbot import ClawBotClient

client = ClawBotClient()
status = client.x.status.get_status()
print(status.is_logged_in)
```

### 2. 读取时间线

```python
from clawbot import ClawBotClient

client = ClawBotClient()
tweets = client.x.timeline.list_timeline_tweets()
for tweet in tweets[:3]:
    print(tweet.id, tweet.text)
```

### 3. AI 发送消息

```python
from clawbot import ClawBotClient

client = ClawBotClient()
response = client.ai.chat.send_message("chatgpt", "say hello")
print(response.content)
```

---

## 七、AI 新增功能的推荐方式

建议 AI 按下面方式扩展，而不是重新写一套脚本：

1. 优先复用 `services/` 里的基础能力
2. 把多步骤流程写进 `workflows/`
3. 把可演示的脚本写进 `examples/`
4. 把验证逻辑写进 `tests/unit` 或 `tests/integration`

这样可以保持类库结构清晰，避免再次回到“测试脚本堆叠式开发”。

---

## 八、一句话总结

复制到其它工程时，真正应被当成产品主体的是 `clawbot/`；其余目录更多是迁移辅助、示例、测试或兼容层。
